# 스프링으로 세션처리하기
1. 세션을 실무 프로젝트에서 여러가지 권한처리할때 많이 활용된다.
	권한에 따른 페이지 접속과 기능버튼 수행기능 여부들을 사용할떄, 많이 사용한다.
2. 위 내용을 처리할떄 DB로 확인하는 방법도 있지만 , 서버 부하에 대한 문제로 위 내용을 session처리할떄가 많다
3. 주요 처리할 내용
	1) 회원정보 테이블로 로그인처리
	2) 로그인 성공시, session설정
	3) session값에 의하여 특정한 화면 처리, 기능 버튼 처리
	4) page별 session 설정 유무에 따른 접근 처리
4. 개발 순서
	1) DB설계 sql작성
	2)dao
	3) service
	 4) controller
         loginFrm.do
         login.do 
          해당 아이디/패스워드가 존재하는 경우, 로그인 성공과 session 설정
         main.do
           메인 페이지에서 session에 따른 출력과 특정권한에 따른 기능처리 ==> ex) 관리자
           session 권한이 없는 경우, 로그인페이지로 이동처리 ==> ex) 회원
